1. 제품 개요 (Overview)

제품명

runwork

한 줄 설명

runwork는 시간을 관리하는 서비스가 아니라,
오늘 내가 멈추지 않고 달려왔다는 사실을 기록하는 서비스다.

제품 유형
	•	개인용 데일리 로그 서비스
	•	실시간 인터랙션 기반 웹 애플리케이션

⸻

2. 문제 정의 (Problem Statement)

대부분의 생산성 도구는 하루를 결과로 평가한다.
	•	얼마나 오래 일했는가
	•	얼마나 많은 일을 끝냈는가
	•	계획을 얼마나 달성했는가

이 과정에서 기록은 곧 평가가 되고,
평가는 부담이 되며,
결국 기록 자체를 포기하게 된다.

문제는 사람들이 게을러서가 아니라,
기록 시스템이 너무 무겁기 때문이다.

⸻

3. runwork의 목적 (Product Purpose)

runwork의 목적은 하루를 관리하는 것이 아니라
하루를 완주했다는 감각을 제공하는 것이다.
	•	완벽하지 않아도 된다
	•	중간에 쉬어도 된다
	•	계획과 달라도 된다

중요한 것은 단 하나다.

“오늘도 나는 멈추지 않았다.”

⸻

4. 제품 철학 및 설계 원칙 (Principles)

4.1 Zero Friction Start
	•	어떤 입력도 없이 바로 RUN 가능
	•	준비 부족이 시작을 막아서는 안 된다

4.2 User Is Always Moving
	•	휴식도 하나의 상태다
	•	멈춤조차 기록의 일부다

4.3 No Retrospective Editing
	•	기록은 수정할 수 없다
	•	하루는 되돌릴 수 없다

4.4 Visual First, Numbers Later
	•	숫자는 보조 정보
	•	흐름과 움직임이 핵심 정보

⸻

5. 제품이 아닌 것 (Non-Goals)

runwork는 다음을 의도적으로 하지 않는다.
	•	타임 트래킹 자동화
	•	생산성 점수 산출
	•	주/월 단위 통계
	•	목표 관리 및 알림
	•	다른 사용자와의 비교

이러한 기능은 v1의 철학을 훼손한다.

⸻

6. 핵심 사용자 경험 (Core Experience)

6.1 전체 사용자 흐름
	1.	Intro – 분위기와 감정 진입
	2.	Entry – 선택 최소화, 즉시 시작
	3.	Run – 실시간 기록과 몰입
	4.	Finish – 하루의 감정적 종결

6.2 Entry Screen
	•	이전 기록 불러오기 (선택)
	•	오늘의 계획 입력 (선택)
	•	아무것도 하지 않아도 RUN 가능

6.3 Run Screen (제품의 핵심)

Run Screen은 runwork의 80%다.
	•	캐릭터가 화면을 가로질러 계속 달린다
	•	배경은 시간과 상태에 따라 변화한다
	•	하단 버튼과 문장 입력으로 상태를 전환한다
	•	화면은 항상 살아 움직인다

사용자는 “기록하고 있다”기보다
“함께 달리고 있다”는 감각을 느껴야 한다.

6.4 Finish Screen
	•	캐릭터가 감속 후 정지
	•	하루의 트랙이 한 번에 펼쳐짐
	•	숫자보다 흐름이 먼저 보임
	•	메시지는 평가가 아닌 응원

⸻

7. 커스터마이징의 역할

커스터마이징은 자유도를 높이기 위한 기능이 아니다.
안정적인 몰입을 위한 사전 설정이다.

허용되는 커스터마이징
	•	버튼 이름 및 개수 (최대 5)
	•	버튼별 배경 테마
	•	캐릭터 선택

제한 사항 (가드레일)
	•	버튼 위치 고정
	•	사용자 에셋 업로드 불가
	•	Run 중 설정 변경 불가

Live Preview
	•	설정 페이지에는 실시간 Run 미리보기가 존재한다
	•	Preview와 실제 Run 화면은 시각적으로 동일해야 한다

⸻

8. 시스템 설계 개요

8.1 Frontend 책임
	•	Run 상태 머신 관리
	•	시간 흐름 및 애니메이션
	•	이벤트 로컬 큐잉
	•	Live Preview 렌더링

8.2 Backend 책임
	•	사용자 설정 검증 및 저장
	•	Day Session 생성 및 종료
	•	이벤트 append-only 저장
	•	요약 데이터 생성

핵심 원칙

시간은 Frontend가 관리하고,
Backend는 사실을 저장한다.

⸻

9. 데이터 및 기록 철학

모든 기록은 **이벤트(Event)**다.
	•	이벤트는 수정되지 않는다
	•	해석은 나중에 한다
	•	요약은 이벤트로부터 계산한다

이 구조는:
	•	자동화 확장 가능
	•	분석 확장 가능
	•	그러나 v1에서는 분석을 하지 않는다

⸻

10. v1 완료 기준 (Definition of Done)

runwork v1은 아래 조건을 만족하면 완료다.
	1.	RUN 버튼으로 즉시 시작 가능
	2.	상태 전환이 즉시 시각적으로 반영됨
	3.	문장 입력으로 로그를 남길 수 있음
	4.	하루를 명시적으로 종료할 수 있음
	5.	해당 하루를 다시 볼 수 있음

이 다섯 가지 외의 기능은 완성 조건이 아니다.

⸻

11. 성공 지표 (Qualitative Success Metrics)

정량 지표보다 정성 지표를 우선한다.
	•	사용자가 하루를 끝내고 앱을 바로 닫지 않는가?
	•	Finish 화면을 끝까지 보는가?
	•	다음 날 다시 RUN을 누르는가?

이 질문에 “그렇다”면, runwork는 성공이다.

더 괜찮은 하루를 만들기 위한 서비스다.

오늘도, runwork.
